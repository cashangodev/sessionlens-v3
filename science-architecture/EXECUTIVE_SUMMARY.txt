KEY ARCHITECTURE DECISIONS
===========================

EMBEDDINGS
  Model:          OpenAI text-embedding-3-small (1536 dimensions)
  Why:            Sweet spot between expressiveness & performance
  Levels:         Moment-level, participant-level (aggregate), session-level, structure profiles
  Similarity:     Cosine distance, capped at 95% confidence

MATCHING (Multi-Level)
  Semantic:       Vector embeddings (50% weight) - captures meaning
  Structural:     10-dimensional phenomenological profiles (30%) - captures lived experience signature
  Metadata:       Demographics, themes, diagnoses (20%) - grounds in context
  Combined:       0.5*semantic + 0.3*structural + 0.2*metadata

CBT SCIENCE
  Framework:      Diagnosis-of-Thought (DoT) prompting - 3 stages
                  1. Subjectivity assessment (fact vs interpretation)
                  2. Distortion scoring (15 CBT categories, confidence 0-1)
                  3. Schema analysis (link to core beliefs)
  Distortions:    15 types (catastrophizing, all-or-nothing, personalization, etc)
  Detection:      Claude streaming for real-time analysis
  Validation:     Clinical advisor review gate

RISK DETECTION (Multi-Layer)
  Layer 1:        Keyword detection (baseline)
  Layer 2:        Negation analysis ("don't want to" → lower score)
  Layer 3:        Temporal analysis (past tense → lower weight)
  Layer 4:        Context analysis (surrounding sentiment, support mentions)
  Result:         Final score = baseline adjusted by context
  Example:        "I want to hurt myself" (0.8) vs 
                  "I don't want to hurt myself" (0.2 after negation adjustment)

SYNTHETIC DATA
  Conservative:   Generate only within constraints
  Methods:        Conditional (expand real participants), 
                  Interpolated (fill embedding space gaps), 
                  Symptom variants (outcome trajectories)
  Quality gates:  >0.75 clinical validity, marked synthetic, human review
  Scaling:        Phase 1 (100 moments, 100% review) → 
                  Phase 2 (500 moments, 20% review) → 
                  Phase 3 (1000+ moments, continuous monitoring)

SAFETY GUARDRAILS
  Hallucination:  All claims grounded in direct quotes
  Confidence:     Capped at 95% maximum (forces humility)
  Diagnosis:      AI detects patterns; only therapists diagnose
  Risk:           Context-aware (understands negation, temporal, surrounding context)
  Clinical:       Clear boundaries - AI ≠ clinician; AI = decision support

DATA MODEL SUMMARY
===================

9 PostgreSQL Tables (all with pgvector support):

1. participants (40 real + up to 9,960 synthetic)
   - Demographics, themes, summary
   - embedding_participant_level (1536-dim aggregate)

2. moments (768 real moments + variants)
   - Verbatim quote, phenomenological classification
   - embedding_moment_level (1536-dim)
   - cognitive_distortions (CBT analysis results)
   - structures_present/absent (10 structures)

3. phenomenological_structures (10 dimensions per moment)
   - body, prereflective, emotion, behaviour, social, cognitive, reflective, narrative, ecological, normative
   - Each: presence (bool) + intensity (0-1)
   - structure_profile (10-dim vector for similarity search)

4. therapy_sessions (uploaded by therapists)
   - Raw transcript, processed JSON
   - moments_extracted, structures_detected, risk_flags
   - matched_experiences (top 10 matches with outcomes)

5. matching_results (cache of all comparisons)
   - semantic_similarity, structural_alignment, metadata_relevance
   - combined_score (weighted), confidence_percentile
   - outcome_status (positive/mixed/neutral/insufficient)

6. cbt_analysis_templates (pre-computed CBT analyses)
   - distortions array with confidence
   - automatic_thoughts array
   - behavioral patterns, intervention suggestions

7. synthetic_data_provenance (track all synthetic generations)
   - generation_method (conditional/interpolated/symptom_variant)
   - clinical_validity_score, diversity_score, realism_score
   - human_review_status, reviewed_by, review_notes

8. risk_detection_log (audit trail for safety)
   - flagged_text, raw_score, context_adjusted_score
   - severity (low/medium/high/critical)
   - clinician_reviewed, clinician_assessment

9. audit_log (complete system audit trail)
   - action, user_id, resource_type, timestamp
   - classification, error handling, performance metrics

WHAT THIS SOLVES
=================

FROM:
  - Keyword matching only (no semantic understanding)
  - 100% hardcoded fake case data (no real learning)
  - No CBT framework (no evidence-based science)
  - Simple keyword risk detection (misses context)
  - Single-level matching (no structural intelligence)

TO:
  - Vector semantics (1536-dim embeddings capture meaning)
  - Real lived experiences (40 real + 9,960 synthetic with quality gates)
  - Rigorous CBT (DoT prompting, 15 distortions, schema analysis)
  - Context-aware risk (negation, temporal, surrounding context)
  - Multi-dimensional matching (semantic + structural + metadata)
  - Clinically-grounded AI (pattern detection, not diagnosis)
  - Responsible synthetic scaling (quality > quantity)

CRITICAL SUCCESS FACTORS
=========================

1. Vector Indexing: HNSW indexing must be properly configured (lists=100 for 10K)
2. CBT Accuracy: Clinical advisor must validate distortion detection on 50+ samples
3. Risk Detection: Multi-layer context analysis MUST handle negation correctly
4. Confidence Calibration: Max 95% confidence enforced in code
5. Clinical Boundaries: Clear separation between AI analysis and therapist decisions
6. Data Quality: Synthetic data must pass clinical validity gate (>0.75)
7. Performance: Semantic search <50ms, full session <60s

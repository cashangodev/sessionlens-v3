KEY TECHNICAL METRICS IN THE DOCS
==================================

Database:
  - 9 PostgreSQL tables
  - 1536-dimensional embedding vectors (text-embedding-3-small)
  - 10-dimensional structure profile vectors
  - HNSW indexing on all vectors (lists=100)

Embeddings:
  - Multi-level: moment, participant, session, structure
  - Model: OpenAI text-embedding-3-small
  - Similarity: Cosine distance (1 - distance for 0-1 scale)

Matching:
  - 3-layer weighting: 50% semantic + 30% structural + 20% metadata
  - Confidence: 0-0.95 (capped to force humility)
  - Top 10 matches returned per situation

CBT:
  - 15 cognitive distortion categories
  - 3-stage DoT prompting (subjectivity → distortions → schemas)
  - Automatic thought identification
  - Behavioral pattern recognition

Risk Detection:
  - 4-layer analysis: keyword → negation → temporal → context
  - Example: "don't want to hurt myself" scores 0.2 (negation adjustment)
  - vs "want to hurt myself" scores 0.8 (no negation)
  - Severity: low/medium/high/critical

Synthetic Data:
  - Conservative methods: conditional, interpolated, variants
  - Quality gates: >0.75 clinical validity
  - Phased: Phase 1 (100 moments), Phase 2 (500), Phase 3 (1000+)
  - All marked synthetic in database

Performance:
  - Embedding latency: ~100ms (OpenAI)
  - Semantic search: <50ms (HNSW)
  - Full session processing: 30-60s (parallel)

Safety:
  - Confidence capped at 95%
  - Risk detection handles context (negation, temporal, surrounding)
  - Hallucination prevention (all claims grounded in quotes)
  - Clinical boundaries enforced (AI ≠ diagnosis)
